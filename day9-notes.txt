# Day Notes

# Topic
Today I practiced a simple IDOR lab using invoice files.

# Authentication vs Authorization
Authentication = proving who the user is
Authorization = checking what the user is allowed to access

# IDOR
IDOR happens when a user changes an ID, number, or file name in the URL and can access data that does not belong to them.

Example:
Ali should access:
 /invoices/15.txt

But if Ali changes it to:
 /invoices/16.txt

And sees Omar's invoice, this is IDOR.

# Lab Setup
mkdir -p ~/idor-lab/invoices
cd ~/idor-lab

echo "Invoice 15 - Owner: Ali - Amount: 20 USD" > invoices/15.txt
echo "Invoice 16 - Owner: Omar - Amount: 90 USD" > invoices/16.txt
echo "Invoice 17 - Owner: Sara - Amount: 150 USD" > invoices/17.txt

# Start Server
python3 -m http.server 8080 --bind 0.0.0.0

# Test URLs
/invoices/15.txt -> 200
/invoices/16.txt -> 200
/invoices/17.txt -> 200
/invoices/18.txt -> 404

# Status Codes
200 = file exists and can be accessed
403 = file exists but access is forbidden
404 = file does not exist

# Important Note
Seeing 200 for many files is not always IDOR.
It becomes IDOR when I am logged in as one user and can access another user's private data.

# Safe Behavior
Ali's invoice -> 200 OK
Omar's invoice -> 403 Forbidden or 404 Not Found

# Loop Practice
for id in 15 16 17 18; do
  echo "Testing invoice $id"
  curl -s -o /dev/null -w "%{http_code}\n" "http://127.0.0.1:8080/invoices/$id.txt"
done

# Loop Meaning
The loop tests multiple invoice numbers automatically and shows only the HTTP status code.

# Final Idea
IDOR = accessing another user's private data by changing an ID, number, or file name because the server does not check authorization properly.
