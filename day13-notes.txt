# Day Notes

# Topic
Today I practiced a mini security assessment on localhost using two open ports.

# Scope
Target:
127.0.0.1

Allowed ports:
8080, 9090

Goal:
Identify open services, verify them manually, and write a simple security report.

# Services Started
Port 8080:
Python SimpleHTTPServer

Command:
python3 -m http.server 8080 --bind 0.0.0.0

Port 9090:
Netcat listener

Command:
nc -lvnp 9090

# Nmap Scan
nmap 127.0.0.1 -p 8080,9090

Service detection:
nmap -sV 127.0.0.1 -p 8080,9090

# Results
8080/tcp open http
Service: SimpleHTTPServer 0.6 (Python 3.13.12)

9090/tcp open zeus-admin?

# Important Note
The ? means Nmap is not fully sure about the service.

Port 9090 was actually a Netcat listener, not confirmed as zeus-admin.

# Manual Verification

For port 8080:
curl -I http://127.0.0.1:8080

Result:
HTTP/1.0 200 OK

For port 9090:
nc 127.0.0.1 9090

I sent:
hello

The message appeared in the Netcat listener, so port 9090 was accepting connections.

# Report Template

Scope:
Target: 127.0.0.1
Allowed ports: 8080, 9090

Finding:
Port 8080 is open and running Python SimpleHTTPServer.
Port 9090 is open and accepting connections.

Risk:
Port 8080 may expose files from the current directory.
Port 9090 is an unknown listener and may allow unexpected connections.

Severity:
Low

Recommendation:
Close both services if they are not needed.
Do not store sensitive files in the shared directory.

Verification:
curl confirmed that port 8080 returned 200 OK.
Netcat confirmed that port 9090 accepted the message "hello".

# Final Idea
Nmap helps find open ports.
-sV tries to identify the service and version.
Manual verification confirms what is really running.
A good report should include Finding, Risk, Severity, Recommendation, and Verification.
