# Introduction to Web Application Security

> Course: Web Application Security Course (Arabic)  
> Lesson: Introduction  
> Topic: Web Application Fundamentals

---

## الهدف من الدرس

فهم أساسيات تطبيقات الويب قبل البدء في اكتشاف الثغرات الأمنية.

---

## ما هو Web Application؟

تطبيق الويب هو برنامج يعمل من خلال المتصفح، ويتواصل مع خادم عبر بروتوكول HTTP أو HTTPS.

أمثلة:

- GitHub
- Gmail
- Facebook
- Amazon

---

## مكونات تطبيق الويب

### 1. Client

هو جهاز المستخدم أو المتصفح.

يشمل:

- HTML
- CSS
- JavaScript

### 2. Server

هو الخادم الذي يستقبل الطلبات ويعالجها ثم يرسل الرد للمستخدم.

### 3. Database

قاعدة البيانات تُستخدم لتخزين المعلومات مثل:

- بيانات المستخدمين
- المنتجات
- التعليقات
- الجلسات

---

## كيف يعمل الطلب؟

1. المستخدم يفتح موقعًا.
2. المتصفح يرسل HTTP Request.
3. الخادم يعالج الطلب.
4. الخادم قد يتواصل مع قاعدة البيانات.
5. الخادم يرسل HTTP Response.
6. المتصفح يعرض النتيجة.

---

## مفاهيم مهمة

| المفهوم | المعنى |
|---|---|
| HTTP | بروتوكول التواصل بين المتصفح والخادم |
| HTTPS | نسخة آمنة من HTTP تستخدم التشفير |
| Request | طلب يرسله المتصفح |
| Response | رد يرسله الخادم |
| Cookie | ملف صغير يخزن بيانات في المتصفح |
| Session | طريقة لتتبع حالة المستخدم بعد تسجيل الدخول |

---

## لماذا هذا مهم في الأمن السيبراني؟

لأن أغلب ثغرات الويب تحدث بسبب أخطاء في:

- تسجيل الدخول
- الصلاحيات
- إدارة الجلسات
- التعامل مع المدخلات
- إعدادات الخادم

## ملاحظاتي

- يجب فهم طريقة عمل الويب قبل تعلم الثغرات.
- كل ثغرة لها سبب منطقي في طريقة معالجة الطلبات.
- التطبيق العملي يجب أن يكون فقط على لابات قانونية أو بيئات أملكها.
