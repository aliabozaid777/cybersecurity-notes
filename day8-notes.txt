Today I learned how URL parameters work and how data can be sent through the URL using a GET request.

# Simple Web Form
I created a small HTML form with an input field and a Send button.

<form action="/" method="GET">
  <input type="text" name="word">
  <button type="submit">Send</button>
</form>

# Start the Web Server
python3 -m http.server 8080 --bind 0.0.0.0

# GET Parameters
When I entered a word in the form, it appeared in the URL.

Example:
?word=cat

This means:
word = parameter name
cat = parameter value

# Multiple Parameters
The & symbol separates more than one parameter.

Example:
/account?id=25&role=user

Path = /account
id = 25
role = user

# Path vs Parameter
Example:
/profile?id=100&view=details

/profile = path
id = 100
view = details

# Important Idea
A user can change parameter values manually in the URL.

Example:
/profile?id=100
/profile?id=101

The server must check whether the user is allowed to access the requested data.

# Authentication vs Authorization
Authentication = confirming who the user is
Authorization = checking what the user is allowed to access

# IDOR
IDOR = Insecure Direct Object Reference

Example:
/invoice?id=15 = my invoice
/invoice?id=16 = another user's invoice

If I can change the ID and see another user's information, this is an IDOR vulnerability.

Another example:
/download?file=report-7.pdf
/download?file=report-8.pdf

If changing the file name lets me download another user's report, this is also IDOR.

# What I Learned Today
- How GET sends data in the URL
- How parameters have names and values
- How to identify the path in a URL
- How multiple parameters are separated with &
- Why changing IDs in URLs can be dangerous
- IDOR is an Authorization problem
