# Day Notes

# Web Server
python3 -m http.server 8000 = start a simple web server

# Localhost vs Local IP
127.0.0.1 = my own device only
192.168.x.x = my device inside the local network

# HTTP Status Codes
200 OK = request successful
301 Redirect = moved to another path
400 Bad Request = wrong request format
401 Unauthorized = login/authentication needed
403 Forbidden = access denied
404 Not Found = page/file not found

# HTTP Headers
curl -I http://IP:8000 = show response headers
Content-Type = type of content
Content-Length = size of response data
Server = server information

# HTTP Methods
GET = request data
POST = send data
PUT = upload/update data
DELETE = delete data

# Cookies and Sessions
Set-Cookie = sent by server to browser
Cookie = sent by browser to server
Session Cookie = used to remember logged-in users

# Authorization
Authorization: Bearer TOKEN = send authentication token in headers
curl -H = send custom HTTP headers

# DNS
nslookup domain.com = get IP address from DNS
MX Record = mail server record
TXT Record = extra DNS information

# Traceroute
traceroute domain.com = show the path packets take to reach a target

# TCP States
LISTEN = service is waiting for connections
ESTABLISHED = active connection exists
TIME-WAIT = connection recently ended

# TCP vs UDP
TCP = reliable, used for websites and file transfer
UDP = faster, used for games, calls, and streaming

# Nmap / Network
nmap IP = scan open ports
nmap -sV IP = detect service version

# Important Concepts
Port = service door
Service = program running on a port
HTTP = not encrypted, usually port 80
HTTPS = encrypted, usually port 443
robots.txt = instructions for search engines, not real protection
Banner Grabbing = collecting service information from server response
