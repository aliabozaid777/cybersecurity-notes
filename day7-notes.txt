# Day Notes

# Web Root
web root = the folder served by the web server

Files inside web root can be accessed from the browser.
Files outside web root are safer and cannot be accessed directly.

# Python Web Server
python3 -m http.server 8080 = start a simple web server
python3 -m http.server 8080 --bind 0.0.0.0 = allow access from other devices on the local network

# Localhost vs Local IP
127.0.0.1 = only my own device
localhost = only my own device
192.168.x.x = my device inside the local network

# curl
curl URL = show the content/body
curl -I URL = show headers and status code only

# HTTP Status Codes
200 OK = file/page exists and request worked
301 Redirect = moved to another path
403 Forbidden = file/page exists but access is denied
404 Not Found = file/page does not exist

# Directory Listing
Directory Listing = the server shows folder contents to visitors

Example:
/backup/ = shows folder contents
/backup/old.txt = shows file content

# robots.txt
robots.txt gives instructions to search engines
robots.txt does not protect files or folders

# Hidden vs Protected
Hidden does not mean protected.
If a file is inside the web root, it may still be accessible if someone knows its name.

Example:
secret.txt may not appear on the homepage, but it can still be opened directly.

# Sensitive Files
Sensitive files should not be inside the web root.

Examples of sensitive files:
config.txt
passwords.txt
backup.zip
database.db

Best practice:
Public files stay inside web root.
Sensitive files stay outside web root.

# Access Control
Access Control = deciding who can access something and who cannot

Example:
/index.html = public
/admin/ = only admin
config.txt = should not be accessible to visitors

# Authentication vs Authorization
Authentication = who are you?
Authorization = what are you allowed to do?

Example:
Login with username/password = Authentication
Accessing admin page as normal user = Authorization problem

# IDOR
IDOR = Insecure Direct Object Reference

Example:
/profile?id=100
/profile?id=101

If I change the ID and see another user's data, this is an Authorization problem.

# Important Security Ideas
Hidden files are not protected files.
403 means the resource exists but access is denied.
404 means the resource does not exist.
200 on a sensitive file means possible information exposure.
robots.txt is not security.
Access Control is one of the most important web security concepts.
